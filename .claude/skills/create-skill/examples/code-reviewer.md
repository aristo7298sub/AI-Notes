# 代码审查指南

## 审查重点

### 1. 代码质量
- 代码是否清晰易读
- 命名是否有意义
- 是否有重复代码
- 函数/方法长度是否合理

### 2. 安全性
- 输入验证
- SQL 注入风险
- XSS 风险
- 敏感信息泄露

### 3. 性能
- 不必要的循环
- N+1 查询问题
- 内存泄漏风险

### 4. 测试
- 是否有对应的测试
- 测试覆盖是否充分
